A production-grade platform that turns an organization's documents into a queryable, **cited**, multi-agent knowledge system — RAG, GraphRAG, agentic AI, guardrails, evaluation, and MLOps, built milestone-by-milestone like a real product.

> Not another RAG chatbot. Every answer is retrieved hybrid (BM25 + dense + graph), reranked, generated by a domain agent, and **verified for citations and groundedness** before it reaches the user.

## Quickstart

```bash
# full stack (Postgres+pgvector, Qdrant, Neo4j, Redis, MinIO, API, web UI)
cp .env.example .env
docker compose up -d          # UI on http://localhost:3000

# or bare development (keyless: in-memory store, extractive answers)
cd backend  && pip install -r requirements.txt && uvicorn app.api.main:create_app --factory
cd frontend && npm install && npm run dev       # http://localhost:5173
```

The platform runs with **zero API keys** (offline embedder + extractive answers); set `OPENAI_API_KEY` in `.env` for synthesized LLM answers.

## What it does

| Capability | How |
|---|---|
| Ingest anything | PDF, DOCX, TXT/MD, HTML, CSV, PPTX + OCR for scans/receipts (layout & table detection) |
| Clean + chunk | normalization, header/footer removal, MinHash dedup; semantic chunking with heading breadcrumbs, whole tables, deterministic chunk ids |
| Index three ways | Qdrant/pgvector vectors + BM25 keywords + Neo4j knowledge graph with evidence on every edge |
| Retrieve | dense · hybrid RRF · cross-encoder/Cohere rerank · **GraphRAG** (finds documents that never name the entity you asked about) |
| Answer | six domain agents (HR, Finance, Research, Developer, Legal, Ops), routed or pinned, streaming over WebSocket with a provenance rail |
| Trust | injection/jailbreak screening, PII masking (Luhn-checked), citation verification, sentence-level groundedness — report attached to every answer |
| Measure | golden-dataset evaluation (precision/recall/MRR/faithfulness/hallucination rate), Prometheus metrics, cost tracking |
| Remember | conversation, user, project, and long-term memory (summarized) |

## Architecture

```
React/TS + Tailwind ── WS/REST ──> FastAPI ──> Guardrails ──> Agent Orchestrator
                                                                │
                        Hybrid retrieval: Qdrant · pgvector · BM25 · RRF · Rerank · Neo4j GraphRAG
                                                                │
                        Ingestion workers: extract → OCR → clean → chunk → embed → graph
```

## Engineering practice

**267 backend tests** (offline — mock transports, fake engines, zero keys) · ruff-clean · strict TypeScript · one feature branch per milestone with `feat:` merges · phase tags `v0.1.0` → `v1.0.0` · CI: lint + tests + typed build + GHCR images on every push to main ([workflow](.github/workflows/ci.yml)).

Remaining scheduled work: **Module 21 — UI/UX Design Review & Professional Interface Enhancement** (light mode, full component polish, admin dashboard), gated on approval of the [design system](docs/design/design-system.md).
